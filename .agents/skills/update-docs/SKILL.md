## 지시사항

방금 완료된 주요 개발 작업을 문서화합니다. 작업 요약: $ARGUMENTS

아래 두 가지 문서를 순서대로 작성하세요.

---

### STEP 1: `/docs/PROJECT_OVERVIEW.md` 업데이트

먼저 다음을 수행하세요:
1. 기존 `/docs/PROJECT_OVERVIEW.md`가 있으면 Read로 읽어 현재 내용을 파악
2. 프로젝트 주요 파일들을 탐색하여 현재 구조와 구현 상태를 파악 (Glob, Read 활용)
3. 아래 형식으로 `/docs/PROJECT_OVERVIEW.md`를 Write (전체 덮어쓰기)

**파일 형식:**
```
# PROJECT OVERVIEW

> 마지막 업데이트: {{오늘 날짜 YYYY-MM-DD}}
> 이 문서는 다른 AI 에이전트가 프로젝트를 빠르게 파악할 수 있도록 작성된 기술 문서입니다.

## 프로젝트 개요
- 앱 이름, 목적, 주요 사용자 시나리오

## 기술 스택
- Framework, 주요 패키지, 버전 등

## 프로젝트 구조
- 핵심 디렉토리와 각 역할 설명
- 중요한 파일 경로와 용도

## 구현된 기능
각 기능에 대해:
- 기능명
- 구현 방식 (어떤 파일, 어떤 패턴)
- 주요 의존성

## 미구현 / TODO
- 아직 구현되지 않은 기능 목록
- 알려진 이슈

## 에이전트를 위한 참고사항
- 이 프로젝트에서 주의할 점
- 코드 컨벤션, 폴더 규칙 등
- 작업 시 알아야 할 특이사항
```

---

### STEP 2: `/docs/{{오늘 날짜 YYYY-MM-DD}}/PROMPTS.md` 생성

오늘 날짜 디렉토리가 없으면 생성 후, 아래 형식으로 PROMPTS.md를 Write하세요.
기존 파일이 있으면 Read 후 내용을 추가(append)하세요.

**파일 형식:**
```
# {{오늘 날짜}} 작업 기록

## 작업 요약
{{$ARGUMENTS}}

## 작업 배경 및 목적
이 작업을 왜 수행했는지, 어떤 문제를 해결하려 했는지 서술

## 주요 의사결정 및 사고 과정
- 왜 이 구조/패턴을 선택했는지
- 고려했던 대안과 선택한 이유
- 어려웠던 부분과 해결 방법

## 구현 상세
- 변경/추가된 주요 파일 목록
- 각 파일에서 한 일

## 다음 작업을 위한 메모
- 이어서 작업해야 할 사항
- 주의사항
```

---

### 완료 후 알림

두 파일 모두 작성이 완료되면, 사용자에게 다음을 간략히 보고하세요:
- 업데이트된 파일 경로 2개
- PROJECT_OVERVIEW.md에서 추가/변경된 주요 섹션
