### S1-1: Drop SVG from allowed uploads (CRITICAL)

**Finding:** `src/middlewares/uploads.ts` accepts SVG files and serves them from the same origin. SVGs execute `<script>` tags when opened in a browser, enabling stored XSS.

**Files to change:**
- `src/middlewares/uploads.ts`

**Changes:**
1. Remove `.svg` from `IMAGE_EXTENSIONS`.
2. Remove the SVG detection branch from `detectMime()`.
3. Remove `'image/svg+xml': '.svg'` from `MIME_EXT` in `AttachmentService`.

```ts
// uploads.ts — remove from IMAGE_EXTENSIONS
const IMAGE_EXTENSIONS = new Set([
  '.jpg', '.jpeg', '.png', '.gif', '.webp',
  '.bmp', '.tiff', '.heic', '.heif', '.avif',
  // '.svg' — REMOVED: SVG executes JS on same origin (stored XSS)
]);

// uploads.ts — remove from detectMime()
// Remove entire SVG block:
// const head = buf.slice(0, 256).toString('utf8').trimStart();
// if (head.startsWith('<?xml') || head.startsWith('<svg')) return 'image/svg+xml';
```

**Acceptance criteria:**
- `POST /attachments/images` with an SVG payload returns 400.
- `POST /attachments/images` with a valid PNG still works.

### S1-3: Fix authentication ordering on upload routes (HIGH)

**Finding:** `app.ts` registers multer middleware directly BEFORE `RegisterRoutes`, so a 10–20 MB file is buffered in memory before JWT auth is checked. Unauthenticated callers can exhaust server memory.

**Files to change:**
- `src/app.ts`

**Changes:**

Remove the two direct `app.post` lines:
```ts
// DELETE THESE:
// app.post('/attachments/images', imageUpload.single('file'), verifyImageMime);
// app.post('/attachments/files', fileUpload.single('file'), verifyFileMime);
```

Add a thin auth guard and re-register before `RegisterRoutes`:
```ts
import { expressAuthentication } from './middlewares/auth';

function requireJwt(req: Request, res: Response, next: NextFunction): void {
  expressAuthentication(req, 'jwt')
    .then((user) => { req.user = user; next(); })
    .catch(() => { res.status(401).json({ message: 'Unauthorized' }); });
}

app.post('/attachments/images', requireJwt, imageUpload.single('file'), verifyImageMime);
app.post('/attachments/files',  requireJwt, fileUpload.single('file'),  verifyFileMime);
```

**Acceptance criteria:**
- `POST /attachments/images` without a JWT returns 401 immediately with no file buffered.
- `POST /attachments/images` with a valid JWT still uploads successfully.

### S1-5: Use `crypto.randomBytes` for invite codes (HIGH)

**Finding:** `BandService.generateInviteCode()` uses `Math.random()`, a non-CSPRNG whose state is reconstructible from observed outputs.

**Files to change:**
- `src/services/BandService.ts`

**Changes:**
```ts
// Before:
function generateInviteCode(): string {
  const alphabet = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
  let out = '';
  for (let i = 0; i < 8; i += 1) {
    out += alphabet[Math.floor(Math.random() * alphabet.length)];
  }
  return out;
}

// After:
import crypto from 'crypto';

function generateInviteCode(): string {
  const alphabet = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
  const bytes = crypto.randomBytes(8);
  let out = '';
  for (let i = 0; i < 8; i += 1) {
    out += alphabet[bytes[i] % alphabet.length];
  }
  return out;
}
```

**Acceptance criteria:**
- `crypto` is already imported in `BandService.ts`... wait, it is not — add import.
- Generated codes look random across many band creates.

## Sprint 2 — Close Auth Leaks (~1 day)

### S2-1: Move WebSocket auth out of URL query string (CRITICAL)

**Finding:** `wss://pagomnini.kro.kr/bands/{id}/chat?token=<JWT>` puts the full JWT in the URL, where it lands in nginx access logs and browser history.

**Files to change:**
- `src/ws/chatServer.ts`
- Frontend (coordinate separately)

**Approach A — Subprotocol header (recommended):**

Client sends: `new WebSocket(url, ['bearer', token])`

Server reads:
```ts
// In handleUpgrade():
const protocols = (request.headers['sec-websocket-protocol'] ?? '')
  .split(',')
  .map(s => s.trim());
const bearerIdx = protocols.indexOf('bearer');
const token = bearerIdx >= 0 ? protocols[bearerIdx + 1] : null;
if (!token) return reject(socket, 401, 'Unauthorized');
```

**Approach B — Short-lived WS ticket (more secure):**
1. Add `POST /auth/ws-ticket` (JWT-authenticated) that creates a 30-second single-use token in the `RevokedToken` table (or a new `WsTicket` table), returns `{ ticket }`.
2. Client gets ticket first, then connects: `wss://…/bands/{id}/chat?ticket=<ticket>`.
3. WS upgrade handler exchanges ticket for userId and marks it used.

**Acceptance criteria:**
- JWT is not present in any query string of a WS URL.
- nginx access log does not show `?token=` for WS upgrade requests.

### S2-2: Add rate limiting (HIGH)

**Finding:** No rate limits anywhere — `/auth/google` is open to credential stuffing; `/bands/join` is open to invite-code brute force.

**Files to change:**
- `package.json`
- `src/app.ts`

**Changes:**


```ts
// src/app.ts
import rateLimit from 'express-rate-limit';

// Auth endpoints: 10 req / 15 min / IP
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 10,
  standardHeaders: true,
  legacyHeaders: false,
  message: { message: 'Too many requests, please try again later.' },
});

  windowMs: 60 * 60 * 1000,
  max: 20,
  standardHeaders: true,
  legacyHeaders: false,
});

// Upload: 30 uploads / min / IP
const uploadLimiter = rateLimit({
  windowMs: 60 * 1000,
  max: 30,
  standardHeaders: true,
  legacyHeaders: false,
});

// Register before RegisterRoutes:
app.use('/auth', authLimiter);
app.use('/bands/join', joinLimiter);
app.use('/attachments', uploadLimiter);
```

**Acceptance criteria:**
- >10 rapid hits to `/auth/google` from same IP return 429.
- >20 rapid hits to `/bands/join` from same IP return 429.

### S2-4: Enforce JWT_SECRET minimum length + shorten token TTL (MEDIUM)

**Finding:** `env.ts` accepts any non-empty `JWT_SECRET`. A short secret is offline-brute-forceable. 7-day default TTL gives attackers a long window.

**Files to change:**
- `src/utils/env.ts`
- `.env.example` (update comment)

**Changes:**
```ts
// src/utils/env.ts
function requiredSecret(key: string, minLength = 32): string {
  const value = required(key);
  if (value.length < minLength) {
    throw new Error(`${key} must be at least ${minLength} characters (got ${value.length})`);
  }
  return value;
}

export const env = {
  // ...
  JWT_SECRET: requiredSecret('JWT_SECRET', 32),
  JWT_EXPIRES_IN: process.env.JWT_EXPIRES_IN ?? '1h', // reduced from 7d
};
```

For full refresh-token support (future sprint):
- Add `RefreshToken` Prisma model (id, userId, expiresAt, usedAt).
- `POST /auth/refresh` exchanges a refresh token for a new access token, rotates the refresh token.
- Refresh TTL: 30 days.

**Acceptance criteria:**
- App throws at startup if `JWT_SECRET` is less than 32 characters.
- Default access token TTL is 1 hour.

```ts
// src/utils/urlValidator.ts
import { env } from './env';

export function requireOwnUrl(url: string | undefined | null, field: string): string | null {
  if (!url) return null;
  const base = (process.env.BASE_URL ?? '').replace(/\/$/, '');
  if (!base) return url; // skip validation if BASE_URL not configured
  if (!url.startsWith(`${base}/static/uploads/`)) {
    throw new BadRequestError(`${field} must point to an uploaded file on this server`);
  }
  return url;
}
```

### S3-2: Scrub token query params from request logs (MEDIUM)

**Finding:** `requestLogger.ts` logs `req.originalUrl` verbatim. Any future endpoint accepting `?token=` will log JWTs.

**Files to change:**
- `src/middlewares/requestLogger.ts`

**Changes:**
```ts
const SENSITIVE_PARAMS = ['token', 'access_token', 'id_token', 'refresh_token'];

function sanitizeUrl(url: string): string {
  return url.replace(
    new RegExp(`([?&])(${SENSITIVE_PARAMS.join('|')})=[^&]+`, 'gi'),
    '$1$2=REDACTED',
  );
}

// In requestLogger:
const line = `${req.method} ${sanitizeUrl(req.originalUrl)} ${res.statusCode} ${ms.toFixed(1)}ms`;
```

**Acceptance criteria:**
- A request to `/?token=abc123` is logged as `GET /?token=REDACTED 200`.
